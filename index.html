Overview of Remote Work Design

This project was built as part of a web development assignment focused on semantic HTML, CSS layout systems, and responsive design. I used this as practice for organizing real websites using a mobile-first approach and GitHub Pages for publishing.

Tools & Workflow

I built this site using tools I currently use in class and workflow practice. The goal was to combine structure (HTML), styling (CSS), and version control (GitHub) into a single published project.

  • VS Code for writing and editing code
  • GitHub Desktop for commits and version control
  • GitHub Pages for publishing the live site
  • CSS Grid for page layout structure
  • Flexbox for navigation alignment

HTML

Used to structure sections like header, navigation, and content areas in a semantic way.

CSS

Used to control layout, spacing, and responsive design across mobile, tablet, and desktop screens.

GitHub

Used to track changes and publish the project through GitHub Pages.

Workspace Setup

I typically build and test this type of project on my MacBook using VS Code and Safari. Most of my workflow involves switching between editing CSS, refreshing the browser, and committing updates through GitHub Desktop. This project helped reinforce how layout systems behave across screen sizes.
